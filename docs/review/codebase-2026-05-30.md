# Codebase Review — Astraa

**Review Date:** 2026-05-30  **Branch:** review/codebase  **Reviewer:** Claude Code (14-agent parallel sweep)

## 1. Executive Summary

Astraa is a well-styled Next 16 / React 19 utility aggregator with healthy TypeScript strictness (`tsc --noEmit` passes clean), a thoughtful animation library, a Zustand storage layer with concurrency-locked IndexedDB, and a strong documented error pipeline — but operationally, large parts of that architecture are wired but never mounted. The single biggest structural risk is that `StoreProvider` is never installed in `app/layout.tsx`, so accessibility-to-DOM sync, periodic activity cleanup, and persisted user preferences are effectively dead code, while a legacy `ActivityProvider` polls fake city data every 3 seconds with zero consumers. The biggest opportunity is consolidation: roughly a dozen confirmed-dead files (animations-showcase, error-boundary, sonner shim, migration shim, several UI primitives) plus 25 unused shadcn primitives with their Radix/recharts/react-hook-form/zod deps can be deleted today for a meaningful bundle and cognitive-load win. The single most surprising finding is that the privacy-first product ships Sentry with `sendDefaultPii: true`, `tracesSampleRate: 1`, full Session Replay, AND a hardcoded CoinGecko API key (`CG-nrVyRWCGSZozXJpLvsBAmQo5`) committed to git and shipped in the client bundle. Test coverage is the weakest dimension (38/100): three of four Zustand stores have zero tests, storage + migration code is uncovered, and there are no component or E2E tests at all on a UI-driven product.

## 3. Critical Findings (must fix)

**StoreProvider is never mounted; the entire Zustand layer is dead at runtime** — Critical — `app/layout.tsx:152-176`, `lib/stores/provider.tsx:14`.
The accessibility-to-DOM sync (reducedMotion / fontSize / highContrast), periodic `clearOldActivities(30)`, performance + memory tracking, and persisted user preferences never run because layout still wraps `ToolsProvider` + `ActivityProvider` instead of `StoreProvider`. Either mount `StoreProvider` in `app/layout.tsx` and drop the legacy providers, or delete the entire `lib/stores/` layer. Pick one — paying the bundle/persistence cost without the behaviour is the worst outcome.

**Two parallel, divergent activity-tracking systems coexist** — Critical — `lib/activity-tracker.tsx:1-153`, `lib/stores/activity-tracking.ts:1-210`, `app/layout.tsx:18,154`.
The mounted Context `ActivityProvider` generates fake `Activity` objects every 3s with hard-coded city strings and synthetic `activeUsers`, and has zero `useActivity()` callers anywhere in the app. The real Zustand store has zero callers too. Delete the legacy provider and its polling interval, or wire consumers to the Zustand store — running both simultaneously costs CPU on every page with no UI benefit.

**CI runs neither lint nor typecheck** — Critical — `.github/workflows/ci.yml:22-26`.
The only job runs `npm test` and `npm run build`. `npm run lint` currently surfaces 15 errors + 9 warnings (24 problems), including 6 React 19 `react-hooks/set-state-in-effect` violations and 2 `Cannot access refs during render` bugs — real correctness issues that would land on `main` today. Add `npm run lint` + `npx tsc --noEmit` steps to CI immediately.

**15 ESLint errors + 9 warnings already live; `--max-warnings 50` invites more** — Critical — `package.json:9` + ESLint output.
Errors include set-state-in-effect in `components/units/unit-converter.tsx:22`, `components/wip.tsx:16`, `components/home/hero-section.tsx:26`, `components/music/pomodoro.tsx:24`, `components/password/password-generator.tsx:61`, `components/ui/floating-navbar.tsx:34`; ref-during-render in `components/landing-background.tsx:59` and `lib/games/memory/useMemoryGame.ts:10`. Fix the 15 errors, then drop `--max-warnings 50` to `--max-warnings 0`.

**Logo uses unconditional `text-white` — invisible in light theme** — Critical — `components/logo.tsx:16`.
The primary brand mark in the landing nav (and via Footer) fails WCAG 1.4.3 and effectively hides the brand on first paint in light mode. Replace `text-white` with `text-foreground`.

**No skip-to-content link; `<main>` only reachable after the full header** — Critical — `app/layout.tsx:156-164`, `components/navigation.tsx`.
`id="main-content"` + `tabIndex={-1}` are set but no `<a href="#main-content">` ever renders. WCAG 2.4.1 (Bypass Blocks) is unmet. Add a visually-hidden-until-focused skip link at the top of `<body>`.

**`ColorDisplay` swatch is a `<div onClick>` with no keyboard handler, role, or tabindex** — Critical — `components/colors/color-display.tsx:34-38`.
Mouse-only, no role, no name — fails WCAG 2.1.1 and 4.1.2. Wrap in a real `<button>` or add `role="button"`, `tabIndex={0}`, and `onKeyDown` for Enter/Space.
